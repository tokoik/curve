# curve - Catmull-Rom Spline サンプルプログラム

## 1. 概要

マウスで Catmull-Rom Spline を描画します。

## 2. ビルド方法

このプログラムは [CMake](https://cmake.org/) を使ってビルドします。CMake のプロジェクト名は bezier なので、生成されるソリューションや実行ファイルの名前もそれに合わせて bezier になります。以下では、ソースコードが置かれているディレクトリで作業する前提で説明します。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行して Visual Studio 2022 用のビルドファイルを生成します。既に build フォルダがある場合は mkdir は不要です。

```bat
cmake -S . -B build -G "Visual Studio 17 2022"
```

1. 生成された build/bezier.slnx または build/bezier.vcxproj を Visual Studio で開きます。

2. CMakeLists.txt で bezier がスタートアッププロジェクトに設定されているので、そのまま「ローカル Windows デバッガー」を実行します。

3. 初回構成時には freeglut が自動取得されるため、ネットワーク接続が必要です。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行して Xcode 用のプロジェクトを生成します。

```sh
cmake -S . -B build -G Xcode
```

1. 生成された build/bezier.xcodeproj を Xcode で開きます。

2. スキームが bezier になっていることを確認して実行します。

3. macOS では GLUT と OpenGL の標準 Framework を使ってビルドします。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 必要なパッケージをインストールします。少なくとも freeglut、GLU、pkg-config、C++ コンパイラが必要です。Ubuntu では例えば次のようになります。

```sh
sudo apt install build-essential cmake pkg-config freeglut3-dev libglu1-mesa-dev
```

1. 以下のコマンドでビルドします。

```sh
cmake -S . -B build
cmake --build build
```

## 3. 使い方

### 3.1 プログラムの起動方法

各 OS とも、ビルド後に生成される実行ファイル名は bezier です。実行すると白いウィンドウが開き、クリックした位置に制御点と Catmull-Rom Spline 曲線が描かれます。

- **Windows**

Visual Studio 上で「ローカル Windows デバッガー」を押して実行できます。コマンドラインから起動する場合の一例は次の通りです。

```cmd
cd build\Debug
bezier.exe
```

- **macOS**

Xcode 上で Run を押して起動できます。アプリケーションバンドルとして生成されるため、Finder から開くこともできます。

- **Ubuntu Linux**

ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

```sh
cd build
./bezier
```

### 3.2 操作方法

- **マウス左ボタンをクリック**: クリックした位置に制御点を 1 つ追加します。追加済みの全制御点を通るように、赤い Catmull-Rom Spline 曲線が再計算されます。

- **マウス右ボタンをクリック**: 追加した制御点をすべて消去し、画面を初期状態に戻します。

- **キーボードの q、Q、または ESC キー**: プログラムを終了します。

### 3.3 表示内容

- 青い点が制御点です。

- 赤い折れ線が、各区間を STEP=10 分割してサンプリングした Catmull-Rom Spline 曲線です。

- 座標系はウィンドウ座標に合わせてあり、クリック位置がそのまま描画位置になります。

## 4. 解説

このプログラムは、OpenGL と GLUT を使って 2 次元平面上に Catmull-Rom Spline を描くサンプルです。ユーザーがマウスで制御点を追加すると、その点列から補間曲線を計算し、青い制御点と赤い曲線を同じ画面上に表示します。描画と入力処理は main.cpp、補間の数式処理は curve.cpp に分かれています。

### 4.1 main() 関数

プログラムの入り口です。GLUT を初期化し、描画、リサイズ、マウス入力、キーボード入力の各コールバック関数を登録してから glutMainLoop() に入ります。以後の処理はイベント駆動で進み、ユーザー入力や再描画要求があるたびに対応する関数が呼ばれます。

### 4.2 init() 関数

OpenGL の初期設定を行います。このプログラムでは頂点配列を有効化し、背景色を白に設定しています。ライティングや奥行き計算は使っておらず、2 次元的な線と点の描画に必要な最小限の設定だけを行っています。

### 4.3 display() 関数

画面の再描画を担当します。最初にカラーバッファをクリアし、登録済みの制御点を青い点として描きます。その後、隣接する制御点の区間ごとに補間パラメータを STEP=10 で等分し、curve() を呼んで曲線上の点列を計算し、それを赤い折れ線として描画します。つまり滑らかな曲線を厳密に 1 本の曲線プリミティブで描くのではなく、多数の短い線分で近似表示しています。

### 4.4 resize() 関数

ウィンドウサイズの変更時に呼ばれます。ビューポートをウィンドウ全体に合わせたあと、glOrtho() で直交投影を設定し、OpenGL の座標系をウィンドウのピクセル座標に近い形へ揃えています。これにより、マウスでクリックした位置をほぼそのまま制御点の座標として使えます。

### 4.5 mouse() 関数

マウスボタン入力を処理します。左ボタンが押されると、クリック位置を z=0 の 3 次元点として point に追加し、同時に key に現在の点番号を追加します。右ボタンが押されると point を空にして画面をリセットします。入力後は glutPostRedisplay() で再描画を要求します。

### 4.6 keyboard() 関数

キーボード入力を処理します。ESC、q、Q のいずれかが押されたときに exit(0) を呼び出して終了します。機能を最小限に絞ったサンプルなので、キーボード操作は終了処理だけです。

### 4.7 curve() 関数と補間アルゴリズム

curve.cpp では、まず catmull_rom() が 1 次元値に対する Catmull-Rom 補間式を計算します。interpolate() はそれを x, y, z の 3 成分へ適用し、1 点の補間座標を求めます。curve() は与えられたパラメータ u がどの区間に入るかを二分探索で求め、その前後 4 点を使って Catmull-Rom Spline を評価します。端点付近では範囲外に出ないよう、先頭または末尾の点を重ねて使う形になっています。

### 4.8 このサンプルの特徴

このサンプルでは key に 0, 1, 2, ... と等間隔のパラメータを入れているため、制御点間の時間軸は一様です。そのため、点の間隔が大きく異なる場合でも同じ重みづけで補間されます。より自然な形状にしたい場合は、制御点間の距離に応じて key を変える chordal や centripetal のパラメータ化へ拡張できますが、現状の実装は Catmull-Rom Spline の基本動作を確認するためのシンプルな構成になっています。
